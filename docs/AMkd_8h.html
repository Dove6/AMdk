enum AMkd_error_code

Error codes set in AMkd_error_info structures.

Only describing critical errors, i.e. these interrupting function execution.
Zero indicating no error and positive values when error is present.

Enumerator
AMKD_ERROR_NONE 

No error.

AMKD_ERROR_MISSING_SETTINGS 

Error: no settings provided for encoding/decoding.

AMKD_ERROR_OUT_OF_MEMORY 

Error: no memory available for allocation for output string.

AMKD_ERROR_MISSING_INPUT 

Error: received NULL as encoded/decoded string.

AMKD_ERROR_INSUFFICIENT_BUFFER 

Error: output buffer allocated by de-/encoding turned out to be too small.

◆ AMkd_error_indicator

enum AMkd_error_indicator

Return code for signaling successful execution or failure of functions supporting error handling.

Only two values are possible:

  • AMKD_FAILURE is returned if any critical error has occured (and, in result, AMkd_error_info.error_code does not equals AMKD_ERROR_NONE).
  • AMKD_SUCCESS is returned otherwise (but still, there can be some warnings set in AMkd_error_info.warning_flags).
Enumerator
AMKD_SUCCESS 

No error.

AMKD_FAILURE 

Error occured.

◆ AMkd_warning_flags

enum AMkd_warning_flags

Binary warning flags set in AMkd_error_info structures.

Describing non-critical errors, signaling suspicious, but correct data or redundancy of operations, etc.
Zero indicating no warnings and consecutive bits (starting from the least significant) meaning different warnings.
According to C language specification, maximum safe number of such flags is sixteen (not counting 0), as enum uses int type and int is guaranteed to be at least 16 bits long.
Flags are to be set by OR-ing previous state of variable with new warning and to be checked by AND-ing the variable with a chosen flag.

Enumerator

No warnings.

AMKD_WARNING_SURPLUS_SETTINGS
